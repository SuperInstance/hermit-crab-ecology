## Method

> *As iron sharpens iron, so one model sharpens another.*

Each model receives:
1. The shell ecology framework (above)
2. A specific shell to inhabit and defend
3. The same base question: what does the architecture look like from your vantage point?

They write independently, then their perspectives are collected here — not merged, not reconciled, but placed adjacent so the *tension* between them becomes the signal.

## The Question

From your shell depth, looking at what SuperInstance has built:
- FLUX bytecode (3 VMs, cross-verified)
- PLATO rooms (5 engine implementations)
- Conservation enforcer (FLUX-enforced budgets)
- Shepherds console (edge deployment)
- The hermit crab architecture itself

**What is the shape of this thing? What does it look like from beyond the intuitive viewpoint? What does it become when it grows?**
